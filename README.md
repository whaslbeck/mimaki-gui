# Mimaki ME-500 GUI

A desktop application for planning, visualising, and transmitting CNC toolpaths to the
**Mimaki ME-500** engraver/router.  Import G-code or HPGL files, arrange objects on the
work area, generate simple shapes directly in the app, and stream the result to the
machine over a serial connection — all without leaving a single window.

---

## Overview

| | |
|---|---|
| **Language** | Python 3.11+ |
| **UI toolkit** | PyQt6 |
| **Machine** | Mimaki ME-500 (483 × 305 mm, MGL-IIC3 protocol) |
| **Platforms** | Linux (Ubuntu 22.04+), macOS (Homebrew Python), Windows (untested) |
| **Serial** | USB-to-serial or RS-232, configurable baud/parity/stop-bits |

The application follows a simple workflow:

1. **Import** one or more G-code or HPGL files (or generate shapes with the built-in wizard).
2. **Arrange** objects on the canvas — move, rotate, scale, mirror, clone, or array them.
3. **Preview** the toolpath in 2D or 3D; filter by Z-depth; run a dry-run animation.
4. **Connect** to the machine, verify the unit resolution, confirm the Z/XY datum, and **send**.

---

## Features

### File Import & Export

**G-code import** reads standard RS-274 G-code as produced by most CAM packages
(FlatCAM, Fusion 360, KiriMoto, etc.).  Supported commands include rapid moves (G0),
linear cuts (G1), absolute/relative positioning (G90/G91), and Z-moves.  Comment lines
(`;` or `(…)`) are preserved and shown as annotations on the canvas.

**HPGL import** reads `.hpgl` / `.plt` / `.hgl` files.  Both `PU`/`PD` pen moves and
`!PZ` Z-axis commands are supported, so files generated by other Mimaki software or
plotter drivers can be reused directly.

**HPGL export** writes a fully self-contained HPGL file for the ME-500, including `IN;`,
`VS` (XY speed), `!VZ` (Z speed), `PU`/`PD` move sequences, and correct `!PZ` depth
commands with a final `PU;` retract.  Speed parameters are taken from the current project
settings.

**Project files** (`.mimaki`) save the complete session state: all imported objects,
their transforms and labels, speed settings, WCS offset, grid configuration, and
forbidden zones.  Projects are stored as JSON and can be version-controlled.  Missing
source G-code files are detected on load with an option to relocate them interactively.
Up to 10 recently opened projects appear in the *File → Recent Projects* menu.

---

### 2D Canvas

The central canvas renders the ME-500 work area (483 × 305 mm) with a configurable grid.

- **Travel moves** (pen-up / G0) are drawn in a distinct colour (default: blue).
- **Cutting moves** (pen-down / G1 at Z < 0) are drawn in red; when *depth colouring* is
  enabled they are colour-mapped from green (shallow) to red (deep) across all Z levels.
- **Comment markers** — small "i" badges — appear at positions in the toolpath where
  embedded G-code comments exist, giving an at-a-glance view of operator notes.
- **Zoom**: mouse wheel or Ctrl+± / Ctrl+=; **Fit all**: Ctrl+0.
- **Pan**: middle-mouse drag or right-drag.
- **Selection**: left-click selects an object; the bounding box and pivot point are
  highlighted.  Right-click opens a context menu (Send, Duplicate, Mirror, Delete, …).

**Z-layer filter** — when a project contains multiple cutting depths, a quick-access
slider and per-depth checklist appear in the *Einstellungen* tab.  Drag the slider to
show only cuts up to a chosen depth, or toggle individual passes via checkboxes.  Useful
for verifying each pass before sending or for diagnosing incorrectly ordered depths.

**Forbidden zones** are user-drawn rectangular regions that represent clamps, fixtures, or
machine limits.  Any attempt to send a job where an object overlaps a forbidden zone is
blocked with a clear error message.  Zones can be drawn (Ctrl+F), renamed, moved, resized,
and deleted, and are saved in the project file.

---

### 3D Preview

Press **Ctrl+3** (or *View → 3D-Vorschau*) to open an interactive 3D visualisation of
the toolpath rendered as an isometric solid.  The view supports mouse rotation, pan, and
zoom; a Z-scale slider exaggerates shallow engravings for easier inspection.  Travel moves
and cutting moves can be toggled independently.

---

### Object Panel

The right-hand *Objekte* tab shows a list of all loaded objects.  Each row has a
visibility toggle, a drag handle for reordering, and the object label.  Selecting an
object opens its properties:

**Transform controls**

| Control | Range | Notes |
|---------|-------|-------|
| Offset X / Y | ±500 mm | Position in work coordinates |
| Scale | 0.001 – 100 × | Uniform; also enter target Width or Height in mm |
| Rotation | −360° – +360° | 0.01° resolution |
| Pivot X / Y | mm | Rotation centre; *Reset Pivot* snaps it back to the object centre |

*Reset Transform* returns all values to their defaults (offset 0/0, scale 1, rotation 0).

**Snap buttons** align the object edge to the work-area boundary:

```
[← X min]  [X max →]    [↑ Y max]  [Y min ↓]
```

**Metadata panel** shows:

- Source file path (truncated, with hover tooltip for the full path)
- Total move count, cutting segments, rapid segments
- Z-depth range and number of distinct passes
- *Kommentare* button → opens a scrollable list of every embedded G-code comment with its
  source line number, for cross-referencing the original CAM program
- *Quellcode* button → opens a read-only, searchable viewer of the original G-code source
  (wrap-around search, monospace font, up to 200 000 lines)

---

### Object Operations

| Operation | Shortcut | Description |
|-----------|----------|-------------|
| Duplicate | Ctrl+D | Copies the selected object 2 mm to the right of its bounding box |
| Clone… | — | Dialog: create 1–200 copies with configurable X/Y spacing |
| Array / Grid… | Ctrl+Shift+A | Fills a rows × cols grid with equal gaps; original becomes [1,1] |
| Mirror horizontal | Ctrl+Shift+H | Flips the toolpath around its vertical centre axis |
| Mirror vertical | Ctrl+Shift+V | Flips the toolpath around its horizontal centre axis |
| Scale to Fit… | Ctrl+Shift+F | Resize to an exact target width or height (aspect ratio preserved) |
| Optimize Order | Ctrl+Shift+O | Reorders all visible objects to minimise total pen-up travel (nearest-neighbour) |
| Delete | Delete | Removes the selected object |

All object operations support **Undo / Redo** (Ctrl+Z / Ctrl+Y).

### Shape Wizard (built-in CAM)

*Edit → Insert Shape…* (Ctrl+I) generates a complete milling toolpath without any
external CAD/CAM software.

**Shape**

- **Rectangle** — enter width and height in mm
- **Circle** — enter radius in mm

**Operation**

- **Contour — outside** (Außenkontur): tool centre path offset outward by the tool
  radius; the cut material dimension matches the drawn rectangle/circle.
- **Contour — inside / cutout** (Innenkontur): tool centre path offset inward; the
  remaining island matches the drawn shape.
- **Pocket / Facing** (Tasche / Planfräsen): fills the area with parallel passes at
  configurable overlap; suitable for surface facing or full pocket removal.

**Parameters**

| Parameter | Description |
|-----------|-------------|
| Tool preset | Select from the global tool library or enter a diameter manually |
| Tool diameter | mm, 3 decimal places (supports 1/8" = 3.175 mm, 1/4" = 6.350 mm, …) |
| Total depth | Total Z plunge in mm (positive input, applied as negative Z) |
| Step per pass | Maximum Z step-down; the wizard calculates the required number of passes |
| Overlap | Pocket step-over as % of tool diameter (5 – 95 %) |

A live preview at the bottom of the dialog shows the move count, cutting segments, and
Z-plunge count.  If the tool is too large for the chosen geometry (e.g. diameter > pocket
width) the wizard shows a warning and disables OK.

Generated shapes are inserted as regular objects and fully support all transform
operations, undo/redo, and HPGL export.

---

### Tool Library

*Machine → Tool Library…* manages global tool presets.  Presets are saved in
`~/.config/mimaki-gui/config.json` and are shared across all projects.

Each preset stores:

| Field | Description |
|-------|-------------|
| Name | Descriptive label (e.g. "V-Bit 0.1 mm", "Flat 3.175 mm") |
| Tool diameter | mm, 3 decimal places |
| XY travel speed | mm/min (rapid, pen-up moves) |
| XY machining speed | mm/min (cutting, pen-down moves) |
| Z travel speed | mm/min (Z retract / approach) |
| Z machining speed | mm/min (Z plunge into material) |

*Apply to Project* copies the preset's speeds into the current project and closes the
dialog.

---

### Speed Settings

The *Einstellungen* tab exposes four speed selectors that directly control the HPGL
`VS` and `!VZ` commands sent to the machine:

| Setting | Typical range | HPGL command |
|---------|--------------|--------------|
| XY travel (PU) | 20 – 80 mm/s | `VS` (global) |
| XY machining (PD) | 0.5 – 50 mm/s | `VS` (global) |
| Z travel | 5 – 30 mm/s | `!VZ` |
| Z machining | 0.5 – 10 mm/s | `!VZ` |

An **estimated job duration** is recalculated immediately when speeds change.

---

### Work Coordinate Offset (WCS)

A global X/Y offset (mm) is added to every coordinate at send time without modifying the
stored toolpath.  This lets you re-register a job to a physical datum on the workpiece
without re-importing or re-transforming objects.

- **Reset** sets both offsets to 0.
- **Bake into Objects** permanently adds the current offset to every object's stored
  position and resets the WCS to 0 (with undo support).

---

### Simulation & Dry-Run

**Dry-run animation** (F4) replays the toolpath on the canvas at adjustable speed (1×,
5×, 20×, 100×, 500×).  The current tool position is drawn as a moving crosshair.  Useful
for verifying the path order and catching obvious errors before committing material.

**Machining simulation** (*Machine → Simulate…*) renders a depth-map of the finished
part:

- Configure workpiece origin (X/Y), width, height, and material thickness.
- Set the tool diameter and render resolution (pixels per mm).
- The simulation sweeps all cutting moves and accumulates Z depth at each pixel.
- Result is displayed as an HSV colour gradient (shallow = green, deep = red).
- **Export STL** saves a 3D mesh of the simulated surface for inspection in any CAD tool.

> Simulation requires NumPy.  If NumPy is not installed the menu item is still visible
> but an informational message is shown on launch.

### Serial Connection & Transmission

**Connecting** — click *Connect* in the transmission panel.  A serial settings dialog
appears (or use the last-used settings).  After the port opens, two machine checks run
automatically:

1. **`OH;`** (work-area query): the machine returns its hard limits in plotter units
   (`minx,miny,maxx,maxy`).  For the ME-500 at 0.01 mm (MGL-IIC3) the expected response
   is `0,0,48300,30500`.  A mismatch triggers a warning to check the unit setting in the
   machine menu — the connection is *not* blocked.
2. **`ZI;`** (machine identification): the response must contain `ME500`.  An unexpected
   identity triggers a warning.

**Pre-send confirmation** — before every job a popup asks the operator to confirm:

- Z axis zeroed to workpiece surface?
- X/Y at origin 0 / 0?

Clicking Cancel aborts without sending anything.

**Send modes**

| Mode | Shortcut | Description |
|------|----------|-------------|
| Send All | F5 | All visible objects in list order |
| Send From… | Shift+F5 | Choose a starting object or move index; preview the start position on canvas |
| Send Selected | Ctrl+F5 | Only the currently selected object |
| Send Bounds | Ctrl+B | Traces the bounding rectangle of all visible objects at the surface (Z=0) — useful for checking workpiece registration |

**During transmission**

- Progress bar shows moves sent / total.
- ETA counts down in real time.
- All sent lines appear in the transmission log (configurable depth, default 500 lines).
- **Pause** (F6) suspends after the current line; **Resume** (F6 again) continues.
- **Stop** (F7) aborts immediately.

**Job log** (*Machine → Job Log…*) lists all past transmissions with timestamp, project
file, move count, duration, and status (finished / stopped / error).

## System Requirements

### Machine

- **Mimaki ME-500** (483 × 305 mm work area)
- Protocol: **MGL-IIC3**
- Plotter unit: **0.01 mm** (verify via machine menu; the app warns on connect if wrong)
- Interface: USB-to-serial (CH340/FTDI/CP2102) or RS-232

### Software

| Dependency | Version | Notes |
|------------|---------|-------|

| PyQt6 | 6.6+ | Auto-installed by `run.sh` |
| pyserial | 3.5+ | Auto-installed by `run.sh` |

| macOS (Homebrew Python) | Tested |
| Other Linux | Should work; use `run.sh` |
| Windows | Untested; PyQt6 and pyserial are available via pip |

## Installation

### Quick start (Linux & macOS)

```bash
git clone https://github.com/whaslbeck/mimaki-gui.git
cd mimaki-gui
./run.sh
```

`run.sh` will:

1. Locate a system Python 3.11+.
2. Create a local `.venv/` virtual environment (no `sudo`, no `--break-system-packages`).
3. Install `PyQt6`, `pyserial`, and `numpy` into the venv via pip.
4. Verify that all application modules import correctly.
5. Launch `main.py`.

On subsequent runs the venv is reused and the package check is fast.

### Check environment without launching

```bash
./run.sh --check-only
```

### Older CPUs (AVX2-incompatible systems)

The PyQt6 wheels from PyPI are built for modern x86-64 CPUs.  On older hardware
(pre-Haswell, some Atom/Celeron systems) the process will crash with `Illegal instruction`
immediately on launch.  Use the system-packages mode instead:

```bash
# Install the distribution-packaged PyQt6 (compiled for a broad CPU baseline)
sudo apt install python3-pyqt6 python3-serial python3-numpy

# Run with system-packages mode — the venv inherits the apt packages
./run.sh --system-packages
```

The `--system-packages` flag creates the venv with `--system-site-packages`, so the
apt-installed libraries are visible inside it.  pip is only used to fill genuine gaps.
Switching between modes automatically recreates the venv if needed.

Both flags can be combined:

```bash
./run.sh --system-packages --check-only
```

### macOS (Homebrew)

```bash
brew install python
./run.sh
```

The venv approach bypasses the PEP 668 `externally-managed-environment` restriction that
Homebrew Python enforces.

### Manual installation (without run.sh)

```bash
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install PyQt6>=6.6 pyserial>=3.5 numpy>=1.24
python main.py
```

## Machine Setup

Before the first use, configure the Mimaki ME-500:

1. **Protocol**: set to *MGL-IIC3* in the machine menu.
2. **Plotter unit**: set to *0.01 mm*.  The app sends `OH;` on connect and compares the
   returned work-area size against the expected `0,0,48300,30500` for the ME-500 at this
   resolution.
3. **Baud rate**: typically 9600 or 19200 — match the setting in *Machine → Serial
   Settings…*.
4. **Flow control**: hardware (RTS/CTS) recommended; adjust the throttle factor in the
   serial settings if the machine buffer overflows.

---

## Configuration

Application settings are stored in `~/.config/mimaki-gui/config.json`.  They persist
across sessions and cover:

- Tool presets (global, shared across all projects)
- Serial port defaults (port name, baud, parity, stop bits, throttle factor)
- Last-used import/project directories
- Recent file list
- UI colours and transmission log line limit

Project-specific settings (speeds, WCS offset, object list, forbidden zones, grid) are
stored in the `.mimaki` project file.
